How TDbeta works - and how to run a tournament without summoning bracket demons

Overview

TDbeta is the control room page. It helps the Tournament Director create match links for scorekeepers, pull names from AYPT signups, build a local bracket, advance matches, export results, and archive the tournament state for later.

The simple mental model is this: signups in, bracket built, matches sent out, winners advanced, results exported.

Before you start

1. Unlock the page

There is a TD code gate at the front. Once unlocked for that browser session, you are in.

2. Decide whether Safe Mode stays on

Safe Mode is the default. It blocks writes and publish actions. That is usually the smart starting point while testing.

3. Set your Tournament ID

Use something consistent like the weekly name and date so your archive makes sense later.

4. Check your table IPs if needed

If you are using local tablet/table calls, make sure the IP list is current before the room gets noisy.

Typical workflow

1

Load or enter players

Either refresh AYPT signups and use that list for seeding, or paste names into the Entrants box manually.

  • AYPT is helpful when players signed up ahead of time
  • manual entry is good for casual or last-minute events
2

Create the bracket

Click New Bracket from Entrants. The app pads the field to the next power of two using BYEs.

  • Double elimination is the default
  • Shuffle can randomize order if you want that behavior
3

Generate match links as matches are called

Use the Match Links tab to enter Player 1, Player 2, table number, ref, and match length. This creates a scoreboard link and QR code.

  • use the QR for quick scanning
  • use the Match Sheet link if you also need the sheet workflow
4

Advance winners in the Brackets tab

You can advance using the dropdown picker or by clicking directly on the player row in the chart.

  • click a player row to mark winner
  • Shift-click a winner to clear it if the downstream path has not started
  • enter scores in the score boxes if you want them shown in the bracket
5

Build and export results

When the event is done, use the Results tab to build placements and export CSV or JSON.

  • CSV is useful for records and posting
  • JSON is useful for backup or future tooling
6

Archive the bracket

Save the current bracket to Archive so the device keeps a local snapshot of the event.

What each tab does

Match Links

Create scoreboard links, QR codes, and match sheet links. This is your “send players to the right place” tab.

AYPT Signups

Pulls in the signup list so you can see who said they are coming and optionally use that list to seed a bracket.

Brackets

Local bracket control center. Build bracket, advance winners, edit scores, and handle latecomer insertion into eligible first-round BYE spots.

Full Bracket Live

Placeholder for live display behavior. In this beta, bracket cloud is disabled, so treat this as future-facing scaffolding.

Results

Build placement output from the current bracket state and export it when the tournament is done.

Archive

Local snapshots of brackets on this browser/device. Great for recovery, reference, or “what happened last week again?” moments.

How latecomers work

TDbeta can place a late-arriving player into an eligible Round 1 BYE path. This helps when somebody shows up after the bracket is already built but before that path has really started feeding forward.

When it works

  • There is a real BYE in Winners Round 1
  • The downstream match has not started yet
  • The player is not already in the bracket

How to do it

  • Type the player’s name in the Late entrant name box
  • Click Add Latecomer
  • The app places them into an eligible Round 1 BYE slot
If you want to increase the odds that a latecomer can be cleanly inserted, starting with an odd number of players usually guarantees at least one first-round BYE somewhere in the bracket.

Practical TD tips

Use Safe Mode first

When testing, leave Safe Mode on until you are sure you want writes enabled.

Name things clearly

Consistent Tournament IDs and player names make archive and results way less confusing later.

Do not over-fix in the moment

If a path already started, it is usually better to keep the bracket honest than to force a heroic edit that creates chaos downstream.

Archive before closing

Local archive is your friend. End-of-night brains are not always your friend.

FAQ

What if I already built the bracket and someone shows up late?

Use the late entrant box if there is an eligible Round 1 BYE path and the downstream path has not started.

What if the next round already started?

That BYE path is effectively locked. At that point, the clean thing is usually to leave the bracket alone and handle the player outside the current draw.

Does this beta publish a live bracket to other screens?

Not right now. Bracket cloud is disabled in this beta, so this page is mainly a local TD control tool.

Can I recover a bracket later?

Yes. Use the Archive tab to save local snapshots, then reload them later on the same browser/device.
